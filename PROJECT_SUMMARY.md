# GEO Promotion Backend 项目总结

## 项目概述

基于GEO（生成式引擎优化）技术的产品推广后端系统，专门为提升产品在AI搜索引擎中的可见性和引用率而设计。

## 核心功能

### 1. 内容优化引擎
- **权威性重写**：增强内容权威性和可信度
- **统计增强**：添加数据支持和证据
- **引用优化**：优化引用和来源
- **结构优化**：改善内容组织和可读性
- **关键词策略**：优化关键词密度和相关性
- **多模态增强**：支持图像、视频等元素描述

### 2. 多平台分发系统
- **国际平台**：OpenAI、Perplexity、Google AI等
- **国内平台**：DeepSeek、豆包、Kimi、通义千问等
- **智能适配**：根据不同平台特性优化内容格式

### 3. 实时监控系统
- **引用监测**：实时跟踪AI平台引用情况
- **性能分析**：计算内容性能指标和分数
- **竞品分析**：分析竞争对手的GEO策略
- **趋势报告**：生成监控报告和优化建议

### 4. 数据分析仪表板
- **可视化报告**：引用趋势、平台分布、效果对比
- **智能建议**：基于数据分析的优化建议
- **竞品洞察**：竞争对手策略分析和应对建议

## 技术架构

### 后端技术栈
- **框架**: FastAPI + Python 3.11+
- **数据库**: PostgreSQL + Redis + ChromaDB
- **AI模型**: OpenAI GPT-4, Sentence Transformers
- **任务队列**: Celery + Redis
- **监控**: Prometheus + Grafana
- **部署**: Docker + Docker Compose

### 核心模块
1. **GEO优化器** (`src/services/geo_optimizer.py`)
   - 实现GEO论文中的6种优化策略
   - 支持批量处理和异步优化

2. **分发服务** (`src/services/distribution_service.py`)
   - 多平台适配器架构
   - 支持同步/异步分发
   - 平台状态监控

3. **监控服务** (`src/services/monitoring_service.py`)
   - 实时引用监测
   - 性能指标计算
   - 竞品分析

4. **API路由** (`src/routers/`)
   - 内容优化API
   - 内容分发API  
   - 效果监控API
   - 数据分析API

## 部署方案

### 开发环境
```bash
# 快速启动
./start.sh dev

# 或手动启动
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn src.main:app --reload
```

### 生产环境
```bash
# Docker部署
./start.sh docker

# 或Kubernetes部署
kubectl apply -f k8s/
```

### 服务架构
```
前端/客户端 → Nginx → FastAPI应用 → 数据库/缓存
                    ↓
              Celery Workers
                    ↓
              外部AI平台API
```
